# 本地 Markdown 笔记软件需求文档

## 一、项目概述

基于 Tauri 2 + Vue 3 + TypeScript 构建的本地笔记管理软件，采用 Markdown 格式存储笔记内容，支持分类管理、标签系统和实时预览。

## 二、界面布局

### 整体布局

采用**左中右三栏布局**，宽度比例为 **1:3:6**：

- 左侧栏（~10%）：分类导航 + 底部设置入口
- 中间栏（~30%）：文件列表 + 标签筛选
- 右侧栏（~60%）：编辑/预览区域

### 响应式设计

- 支持窗口缩放时按比例自适应
- 最小窗口宽度限制，确保三栏可用性

## 三、功能模块

### 3.1 左侧栏 - 分类管理

| 功能 | 描述 |
|------|------|
| 分类列表 | 显示所有笔记分类（如：技术、生活、工作） |
| 新建分类 | 点击按钮创建新分类 |
| 重命名分类 | 右键菜单或双击编辑分类名称 |
| 删除分类 | 右键菜单删除，**连同文件夹及文件一并删除**，进入系统回收站 |
| 分类选中 | 点击分类，中间栏显示该分类下的文件 |
| 设置入口 | 左下角显示设置图标，点击打开设置弹窗 |

### 3.2 中间栏 - 文件列表

| 功能 | 描述 |
|------|------|
| 文件列表 | 显示当前选中分类下的所有 Markdown 文件 |
| 标签显示 | 每个文件项显示关联的标签 |
| 标签筛选 | 顶部显示标签列表，点击可筛选文件 |
| 新建文件 | 创建新的 `.md` 文件 |
| 重命名文件 | 右键菜单或双击编辑文件名 |
| 删除文件 | 右键菜单删除，**进入系统回收站** |
| 文件排序 | 支持按修改时间/创建时间/名称排序 |
| 文件选中 | 点击文件，右侧显示内容 |

### 3.3 右侧栏 - 编辑与预览

| 功能 | 描述 |
|------|------|
| 源码编辑 | 默认显示 Markdown 源码，支持编辑 |
| 标签编辑 | 编辑区域顶部可添加/删除标签 |
| 实时保存 | 编辑时自动保存 |
| 预览模式 | 点击"预览"按钮切换到渲染视图 |
| 模式切换 | 预览/源码模式一键切换 |
| 语法高亮 | 编辑时支持 Markdown 语法高亮 |

### 3.4 设置弹窗

| 设置项 | 描述 |
|------|------|
| 笔记目录 | 配置笔记存储的根目录，支持选择本地文件夹 |
| 自动保存 | 开启/关闭自动保存功能 |
| 保存间隔 | 自动保存的时间间隔（秒） |
| 编辑器字号 | 设置编辑器字体大小 |
| 主题模式 | 明亮/暗黑/跟随系统 |

**设置界面设计**：

- 模态弹窗，点击遮罩层可关闭
- 分组展示各设置项
- 修改后自动保存到本地配置文件

## 四、标签系统

### 4.1 标签管理

| 功能 | 描述 |
|------|------|
| 添加标签 | 为笔记添加一个或多个标签 |
| 删除标签 | 从笔记移除标签 |
| 创建标签 | 输入新标签名自动创建 |
| 标签颜色 | 支持为标签设置颜色（可选） |

### 4.2 标签存储方案

在 Markdown 文件顶部使用 Front Matter 存储标签：

```markdown
---
tags: [技术, Vue, 前端]
created: 2026-03-06
modified: 2026-03-06
---
# 笔记标题

正文内容...
```

### 4.3 标签筛选

- 中间栏顶部显示当前分类下的所有标签
- 点击标签可筛选包含该标签的文件
- 支持多标签组合筛选（AND/OR 逻辑可选）

## 五、数据存储

### 5.1 存储结构

```
{笔记目录}/
├── 技术/
│   ├── 学习笔记.md
│   └── 项目文档.md
├── 生活/
│   └── 日记.md
└── 工作/
    └── 会议记录.md
```

### 5.2 配置文件

存储位置：`{用户数据目录}/firefly-notes/config.json`

```json
{
  "notesDir": "D:/Documents/Notes",
  "autoSave": true,
  "saveInterval": 3,
  "editorFontSize": 14,
  "theme": "system"
}
```

### 5.3 存储方案

- 每个分类对应一个文件夹
- 每篇笔记为独立的 `.md` 文件
- 标签信息存储在 Markdown Front Matter 中
- 使用 Tauri 文件系统 API 进行读写
- 首次启动时引导用户选择/创建笔记目录

## 六、删除策略

| 场景 | 行为 |
|------|------|
| 删除分类 | 连同文件夹及内部所有文件移入系统回收站 |
| 删除文件 | 单个文件移入系统回收站 |

**技术实现**：使用 Rust 的 `trash` crate 将文件移入系统回收站，支持跨平台（Windows/macOS/Linux）。

## 七、技术方案

| 层级 | 技术选型 |
|------|----------|
| 框架 | Tauri 2 + Vue 3 |
| 编辑器 | CodeMirror 6（轻量、可扩展） |
| Markdown 渲染 | markdown-it 或 marked |
| Front Matter 解析 | gray-matter 或 front-matter |
| 样式 | Tailwind CSS 4 |
| 状态管理 | Pinia |
| 文件操作 | Tauri fs API |
| 目录选择 | Tauri dialog API |
| 回收站 | Rust trash crate |
